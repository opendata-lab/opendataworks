# 前端开发与设计标准指南

本指南定义了数据智能体控制台、管理后台、数据看板、配置工作台以及数据可视化（尤其是 `opendataagent` 及其相关 SaaS 产品）的前端设计规范与技术栈要求，确保界面呈现专业、严谨、高信息密度且易于落地。

---

## 技术栈规范 (Stack Mandate)

新前端页面和模块应遵循以下标准技术栈：

- **开发框架**: Vue 3 (使用 Composition API 与 `<script setup>`)
- **样式引擎**: Tailwind CSS (基于清晰的设计 Token 进行实用类组合)
- **数据可视化**: ECharts (用于数据图表、趋势和运行监控视图)
- **图标库**: `lucide-vue-next` (仅使用 light 或 regular 字重)

---

## 视觉与设计标准 (Visual Constraints)

界面风格应保持专业、克制、清晰且易读，避免无意义的视觉噪点。

### 1. 颜色与表面 (Color and surfaces)
- **应用背景**: 使用浅灰色 `bg-gray-50` 或 `bg-[#F9FAFB]`。
- **主内容容器**: 卡片、表格、弹窗等使用白色 `bg-white`。
- **主标题文本**: 使用深色 `text-gray-900`。
- **正文及描述**: 使用中灰色 `text-gray-600`。
- **禁用或辅助文本**: 使用浅灰色 `text-gray-400`。
- **结构分割线**: 使用细线 `border-gray-200`。
- **商务蓝强调色**: 仅在主要操作、激活状态和重点高亮处使用，如 `text-blue-700`、`bg-blue-700` 或 `bg-blue-50`。
- **限制使用渐变**: 默认界面不依赖刺目的渐变色或过饱和的色彩填充。

### 2. 字体与排版 (Typography)
- **默认字体**: 无衬线字体，优先使用 `Inter` 或系统默认 UI 字体。
- **字号等级**: 使用 Tailwind 标准字号，如 `text-sm`、`text-base`、`text-lg`。
- **行高控制**: 段落与详细描述使用 `leading-relaxed` 或 `leading-loose`。
- **字距微调**: 较大标题使用 `tracking-tight` 以提升视觉紧凑度与精致感。

### 3. 布局与间距 (Layout and spacing)
- **栅格系统**: 基于 8px 基础网格。
- **常用间距**: 使用 Tailwind 常用间距如 `2 (8px)`、`4 (16px)`、`6 (24px)`、`8 (32px)`、`12 (48px)`、`16 (64px)`。
- **标准布局间隔**: 卡片或列表间距首选 `gap-4` 或 `gap-6`。

### 4. 圆角与投影 (Shape and depth)
- **圆角**: 容器圆角首选 `rounded-md` 或 `rounded-lg`。除小型 Badge 标签外，避免过多使用胶囊状 (Pill) 圆角。
- **投影**: 卡片默认使用微弱投影 `shadow-sm`；下拉菜单、气泡框等浮层使用 `shadow-md`。避免使用重度阴影或暗色霓虹发光效果。

---

## 数据可视化规范 (Data Visualization)

图表是数据中台的重要组成部分，需满足以下要求：

1. **响应式自适应**: 确保所有 ECharts 图表容器能够跟随窗口或父容器宽度自适应缩放。
2. **主题配色统一**: 覆盖 ECharts 默认配色，使用上述商务蓝与灰色系等专业企业级调色板。
3. **极简坐标轴**: 减少非必要的网格线、图例边框与轴线，仅保留关键的数值标签。
4. **高可读 Tooltip**: 悬浮提示框应清晰易读，提供明确的业务上下文度量单位。
