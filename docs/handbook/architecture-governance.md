# 架构治理本体与口径规范指南

本指南定义了架构治理领域的业务语义、本体模型关系、指标口径以及语义交接规范，用于规范系统对组件、微服务、数据库、Kafka 等架构对象的解释和口径对齐。

---

## 边界与职责

- **业务语义职责**: 负责将用户词汇稳定映射到本体对象、关系、默认口径和本体关联查询函数。
- **不涉及的职责**: 不负责具体的 SQL 生成方法、验证、执行，也不涉及工具选择（如 Portal MCP、`run_sql.py` 等）。
- **交接契约 (`handoff`)**: 当用户需要查询真实数据时，应输出 `query_functions` 作为语义交接契约，并流转到 SQL 问数和平台工具链，而非直接将 `handoff` 面向用户作为最终数据答案。

---

## 核心本体与关联查询

默认产出三要素：**本体 (Ontology)**、**口径 (Caliber)**、**关联查询函数 (Query Functions)**。

### 1. 语义映射与对象定位
- 意图识别与语义交接规范，参考本体库中的对象、属性、函数与关系定义。
- 快照数据统计时，默认使用最新可用快照，即 `MAX(ds) <= target_date`。
- 未指定环境时，默认使用生产环境 (`PROD`)。

### 2. 口径定义硬规则

- **本体优先原则**: 不得绕过已有本体定义去猜测对象、关系或口径。
- **组件统计口径**:
  - 用户提及“组件数”且无特殊说明时，默认映射为“等级保障组件” (`graded_assurance_component`)，而非全量组件。
  - 用户提及“微服务数”时，默认映射为“等级保障微服务组件” (`graded_assurance_microservice_component`)。
- **调用与依赖区别**: 声明依赖不代表真实调用；必须满足“近 6 个月有真实调用记录” (`latest_invoke IS NOT NULL`) 才判定为真实调用。
- **数据库资产区分**: 必须严格区分数据库类型、数据库实例名、Schema 名和表名。用户明确提供 Schema 时，不得直接当作 `database_name` 处理。
- **Kafka Topic 使用判定**: 必须同时覆盖生产者与消费者两个方向，结合 6 个月窗口表进行统计：
  - 生产者来源: `tech.dws_tech_ops_cmp_kafka_producer_call_cnt_6m`
  - 消费者来源: `tech.dws_tech_ops_cmp_kafka_consumer_call_cnt_6m`
- **风险统计口径**: 明确区分架构风险、优化建议与季报风险口径，并对 `core-service-depend-common` 类型的依赖关系进行正确归属。

## 异常与限制处理

- **权限受限场景**: 涉及敏感、涉密、受限系统或调用关系明细时，在未确认用户授权前不输出明细、不绕过权限限制。只提供权限说明和脱敏/申请路径。
- **工具失败兜底**: 用户反馈接口超时或执行失败时，说明证据不足，不给出确定性结论，建议其重试、缩小时间范围或使用异步分析。
