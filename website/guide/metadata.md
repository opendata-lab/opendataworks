# 元数据管理

OpenDataWorks 提供完整的数据资产元信息管理能力，支持从 ODS 到 ADS 的全链路分层管理。

## 数据分层

| 层级 | 说明 | 典型用途 |
|------|------|----------|
| ODS | 操作数据层 | 原始数据接入，保持源系统结构 |
| DWD | 明细数据层 | 数据清洗、标准化、去重 |
| DIM | 维度层 | 维度表管理（地区、时间、产品等） |
| DWS | 汇总数据层 | 轻度聚合，面向主题域 |
| ADS | 应用数据层 | 面向业务应用的宽表和指标 |

## 表管理

### 创建数据表

在「数据资产」模块中创建表，需要指定：

- 表名称和中文描述
- 所属数据层级
- 所属数据域和业务域
- 存储引擎（MySQL、Doris 等）

### 字段管理

每张表支持维护字段级元信息：

- 字段名称和数据类型
- 字段注释
- 是否主键 / 是否允许为空
- 字段排序

## 数据域

数据域用于对数据表进行业务维度的分类管理，例如：

- 用户域
- 交易域
- 商品域
- 营销域

通过数据域可以快速定位和筛选相关数据表。

## 生命周期

数据表支持生命周期状态管理：

- **开发中** — 表结构设计阶段
- **已上线** — 正式使用中
- **已下线** — 不再更新，保留历史数据
- **已归档** — 数据已归档，可清理
