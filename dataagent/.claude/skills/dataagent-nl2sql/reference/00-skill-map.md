# 技能地图

先结论：本 Skill 只决定通用问数方法和找数路径；业务术语、指标口径、本体、别名和歧义消解由已启用的业务知识 Skill 负责，真实平台能力由 OpenDataWorks Platform Tools Skill 负责。

## 问题类型到方法路径

| 问题类型 | 先确认 | 方法重点 | 默认输出 |
| --- | --- | --- | --- |
| 统计 | 指标、过滤、时间范围、目标表 | 确认口径后定位候选表和字段，生成聚合查询方案 | 表格 |
| 对比 | 指标、分组维度、共同过滤、时间范围 | 确认同一口径和同一过滤范围，选择分组字段 | 条形图或表格 |
| 趋势 | 指标、时间字段、时间粒度、时间范围 | 确认时间字段和粒度，避免按相似字段猜测 | 折线图或表格 |
| 占比 | 指标、分类维度、类别数量 | 确认分类字段和分母口径 | 饼图或表格 |
| 明细 | 明细对象、字段、过滤、排序、行数 | 确认展示字段、排序字段和行数限制 | 表格 |
| 诊断 | 目标对象、所需证据类型、范围 | 优先收集 metadata、DDL、血缘或数据源证据 | 表格和结论 |
| 术语或口径 | 业务知识是否已定义 | 交给业务知识 Skill | 中文解释 |
| SQL 示例 | 语义、库表、字段、约束 | 本 Skill 只做 SQL 前检查和方法约束 | SQL 模板说明 |

## 快速判断规则

- 问“多少、数量、总数、次数”通常是统计。
- 问“各类、各状态、各对象”通常是对比或占比。
- 问“最近、按天、按周、变化、趋势”通常是趋势。
- 问“明细、列表、最近记录”通常是明细。
- 问“上游、下游、血缘、DDL、建表语句、路由到哪里”通常是诊断，需要平台工具证据。
- 问“是什么、怎么算、口径、别名”先走业务知识 Skill。

## 先追问的情形

- 业务术语或指标口径没有被用户或业务知识 Skill 确认。
- 目标 database、schema 或 table 不唯一。
- 时间范围或时间粒度不清。
- 对比问题没有指定分组维度。
- 趋势问题没有指定指标或时间字段。
- 候选表跨不同数据源。
- 查询需要 JOIN、去重、明细定位或血缘映射，但关联键不清。

## 何时使用平台工具 Skill

- 需要通过数据源、表名、表注释、字段名、字段注释找到候选表或相似表。
- 需要确认 DDL、血缘、字段详情、数据源或只读执行结果。
- 已形成 SQL 方案，需要真实验证或执行。
- 需要把结果转换为前端可渲染的图表契约。

## 何时停止

- 第一次真实结果已经能回答问题。
- 第一次图表结果已经匹配结果形状。
- 工具返回不可重试失败，并且失败原因足以向用户说明缺口。
