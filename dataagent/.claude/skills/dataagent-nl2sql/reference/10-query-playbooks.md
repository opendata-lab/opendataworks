# 场景 Playbooks

先结论：所有 playbook 都先完成业务语义确认，再进入 metadata / datasource / SQL。这里不定义任何业务术语或指标，只定义通用方法。

## 通用 SQL 前检查

执行 SQL 前确认：

- 目标：用户到底要统计、对比、趋势、占比、明细还是诊断。
- 语义：指标公式、业务术语、别名、例外规则已由用户或业务知识 Skill 确认。
- 数据位置：database/schema、engine、table、必要字段已确认。
- 过滤：时间范围、状态、对象范围、维度/粒度已确认。
- 安全：只读、schema 前缀、LIMIT、单源路由。
- 关联：JOIN、去重、明细定位、血缘映射所需主键、唯一键或关联键已确认。

## 统计

1. 确认指标口径和过滤条件。
2. 确认目标表和必要字段。
3. 生成聚合 SQL。
4. 先验证，再执行。
5. 结果为空时说明口径和空结果，不换表试探。

## 对比

1. 确认对比维度和指标。
2. 确认所有分组使用相同时间范围、过滤条件和口径。
3. 生成分组聚合 SQL。
4. 结果适合图表时生成条形图契约；否则保留表格。

## 趋势

1. 确认指标、时间字段、时间粒度和时间范围。
2. 生成按时间粒度聚合的 SQL。
3. 时间字段不唯一时，先依据业务知识或 metadata 确认；仍不清楚则追问。
4. 结果适合图表时生成折线图契约。

## 占比

1. 确认分类维度、指标和类别范围。
2. 类别过多时优先表格或条形图，不强制饼图。
3. 生成分类聚合 SQL。
4. 结果适合且类别数量有限时生成饼图契约。

## 明细

1. 确认明细对象、展示字段、过滤条件和排序。
2. 必须带 LIMIT。
3. 避免 `SELECT *`，除非用户明确要预览且字段未知。
4. 不强行生成图表。

## 诊断

1. 确认目标对象和证据类型：metadata、lineage、DDL、datasource 或 SQL result。
2. 优先使用专用证据工具。
3. 只有专用证据缺少必要字段时，才补充只读 SQL。
4. 首个足够证据返回后收口。

## 术语或口径

1. 先使用业务知识 Skill。
2. 本 Skill 只在语义确认后提供 SQL 方法检查。
3. 纯解释问题通常不执行 SQL。
